# Vessel Usage Guide

## Getting Started

Building Vessel:
```bash
./scripts/setup.sh  # Or manually via cmake/make
```

### Initializing a Project
Run `vsl init` in any directory where your code lives. This creates a `vessel.json` file.

### Packaging an App
Run `vsl pack` to compile and bundle your app. It will detect your `CMakeLists.txt` automatically. The output file will be named `<app_name>.vsl`.

**Advanced Options:**
- `--keep`: Preserves the intermediate structure folder (`<name>.vsl_struct`) for inspection and debugging. This allows you to verify exactly which libraries and assets were included before compression.

### Distributing Your Bundle
Share the `.vsl` file with anyone on a modern Linux distro. No installation of Vessel is required for the user.

## CLI Commands

### `update`
Cleans up the registry and checks for updates. This command is useful if you want to ensure all your registered apps are still accessible.

### `remove <name>`
Completely uninstalls an app that was registered by Vessel. This deletes:
- The registry entry.
- The installed `.vsl` bundle in `~/.local/share/vessel/bin`.
- Desktop entries in `~/.local/share/applications` and `~/Applications`.
- Associated icons in `~/.local/share/icons`.
- `bin/`: Contains the executable specified by `bin_file`.
- `lib/`: All necessary library dependencies.
- `res/`: Bundled resources and the application icon.
- `vessel.json`: The manifest specifying the application metadata and entry point.

## Manifest Configuration Reference

The `vessel.json` file allows for extensive customization:

- `name`: (string) The "Fancy Name" used for the desktop shortcut and system integrations.
- `bin_file`: (string) The name of the binary generated by your build process. This file is copied into the bundle and launched at runtime.
- `version`: (string) Semantic versioning of your app.
- `icon`: (string) Path to the application icon. This will be used for the desktop shortcut.
- `build_dir`: (string) The directory where your build is orchestrated (defaults to `build`).
- `build_cmd`: (string) The custom command used to build your project (defaults to `cmake .. && make`).
- `dist_dir`: (string) The output directory for the final `.vsl` bundle (defaults to `.`).
- `includes`: (array of strings) A list of paths (directories or files) to include in bundle.

Example `vessel.json`:
```json
{
  "name": "Fancy Pong",
  "bin_file": "pong",
  "version": "1.5.0",
  "description": "Full-screen multi-word titled Pong clone.",
  "icon": "assets/icon.png",
  "build_dir": "custom_build",
  "build_cmd": "make -j8",
  "dist_dir": "bin",
  "includes": [
    "assets/audio",
    "assets/textures"
  ]
}
```
